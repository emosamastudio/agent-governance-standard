# Shared Operating Model

1. Favor long-term correctness, governance, and maintainability over speed.
2. Before planning or stack selection, clarify whether the delivery target is a final-system build or a fast MVP.
3. Do saturated planning before parallel execution.
4. Keep the mainline explicit and protect it from sideline drift.
5. Require documents, logs, hooks, and review as part of the system.
6. Define acceptance before implementation and before completion.
7. Re-check technology choices when cognition updates.
8. Use commits as process evidence, not only as a final packaging step.
