# Technology Selection and Cognitive Freshness

1. Technology choices must be made after the delivery intent is explicit:
   - if the user chose a final-system build, optimize for the final system
   - if the user chose a fast MVP, optimize for fast validated learning without pretending the stack is the final architecture
2. During selection, perform live cognition checks:
   - what may be outdated
   - what changed recently
   - what alternatives now exist
   - what the current limitations are
3. For critical technical points, run explicit research before locking the stack.
4. If research updates the mental model, revisit the architecture and stack choice.
5. If the current proposal conflicts with the recorded delivery intent, revise it before proceeding.
6. Capture meaningful cognition updates as reusable assets when they will matter again.
